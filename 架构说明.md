## 组件说明

### 1. Gateway Service (API 网关)

- **端口**: 8080
- **职责**:
  - 统一入口，所有外部请求通过网关路由到后端微服务
  - JWT 令牌验证，保护后端服务
  - 路由规则配置（基于 Nacos 配置中心）
  - 负载均衡（lb:// 协议）
- **路由规则**:
  - `/api/users/**` -> `user-service`
  - `/api/orders/**` -> `order-service`

### 2. User Service (用户服务)

- **端口**: 8081
- **职责**:
  - 用户 CRUD 操作
  - 提供用户信息查询接口
- **技术栈**: Spring Boot + MyBatis + MySQL (user_db)
- **数据库表**: `user`

### 3. Order Service (订单服务)

- **端口**: 8082
- **职责**:
  - 订单 CRUD 操作
  - 通过 OpenFeign 调用 user-service 获取用户信息
  - 创建订单时验证用户是否存在
- **技术栈**: Spring Boot + MyBatis + MySQL (order_db) + OpenFeign
- **数据库表**: `orders`
- **Feign 客户端**: `UserFeignClient` -> user-service

### 4. Nacos (服务注册与配置中心)

- **端口**: 8848, 9848, 9849
- **职责**:
  - 服务注册与发现
  - 动态配置管理（配置中心）
  - 服务健康检查

### 5. MySQL (数据库)

- **端口**: 3306
- **数据库**:
  - `user_db` - 用户数据
  - `order_db` - 订单数据

### 6. Jenkins (CI/CD)

- **端口**: 9090 (映射容器内 8080)
- **职责**:
  - 代码拉取与编译
  - Docker 镜像构建与推送
  - 自动化部署

## 服务调用链路

```
客户端请求
    |
    v
Gateway (JWT 鉴权 + 路由)
    |
    +-- /api/users/** --> User Service --> MySQL (user_db)
    |
    +-- /api/orders/** --> Order Service --> MySQL (order_db)
                                   |
                                   +-- Feign --> User Service
```

## 关键设计决策

1. **Nacos 替代 Eureka**: 同时提供服务发现和配置中心功能，减少基础设施数量
2. **OpenFeign 替代 RestTemplate**: 声明式 HTTP 客户端，与服务发现集成更好
3. **JWT 网关鉴权**: 集中式认证，白名单路径（/actuator、/api/auth）跳过验证
4. **Feign 降级**: UserFeignClientFallbackFactory 提供熔断降级能力
5. **Nacos 配置热更新**: 通过 `spring.config.import` 动态加载配置
