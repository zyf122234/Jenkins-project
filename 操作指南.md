# 操作使用说明文档

## 一、环境要求

### 1.1 基础环境

| 软件 | 版本要求 | 说明 |
|------|---------|------|
| JDK | 11+ | Java 开发工具包 |
| Maven | 3.6+ | 项目构建工具 |
| Docker | 20.10+ | 容器化平台 |
| Docker Compose | 2.0+ | 容器编排工具 |

### 1.2 检查环境

```bash
# 检查 Java 版本
java -version

# 检查 Maven 版本
mvn -version

# 检查 Docker 版本
docker --version

# 检查 Docker Compose 版本
docker-compose --version
```

## 二、项目构建

### 2.1 编译项目

```bash
cd 微服务Jenkins-Docker项目实践

# 编译整个项目（跳过测试）
mvn clean package -DskipTests

# 编译完成后，每个服务的 target 目录下会生成 JAR 包
# gateway-service/target/gateway-service-1.0.0.jar
# user-service/target/user-service-1.0.0.jar
# order-service/target/order-service-1.0.0.jar
```

### 2.2 构建 Docker 镜像

```bash
# 方式一：使用 docker-compose 构建
docker-compose build

# 方式二：单独构建某个服务
docker build -t microservice-demo/gateway-service:1.0.0 ./gateway-service/
docker build -t microservice-demo/user-service:1.0.0 ./user-service/
docker build -t microservice-demo/order-service:1.0.0 ./order-service/
```

## 三、两种运行方式

### 3.1 本地开发运行（仅启动基础设施）

适用于本地开发调试，只启动 Nacos 和 MySQL，微服务在 IDE 中运行：

```bash
# 只启动 Nacos 和 MySQL
docker-compose up -d nacos mysql

# 等待服务就绪后，在 IDE 中启动各微服务
# 默认 profile (dev) 连接 localhost:3306 和 localhost:8848
```

### 3.2 Docker 全容器运行（推荐部署方式）

适用于完整部署，所有服务均在 Docker 容器中运行：

```bash
# 启动所有服务
docker-compose up -d

# docker-compose 会自动使用 docker profile
# 微服务连接容器名: mysql:3306, nacos:8848
```

## 四、服务启动

### 4.1 一键启动（推荐）

```bash
# 启动所有服务
docker-compose up -d

# 查看服务状态
docker-compose ps

# 查看服务日志
docker-compose logs -f

# 查看某个服务日志
docker-compose logs -f gateway-service
docker-compose logs -f user-service
docker-compose logs -f order-service
```

### 4.2 分步启动

```bash
# 第一步：启动基础设施（Nacos + MySQL）
docker-compose up -d nacos mysql

# 等待 Nacos 和 MySQL 启动完成（约 30 秒）
docker-compose logs -f nacos  # 看到 "Nacos started successfully" 表示启动成功

# 第二步：启动微服务
docker-compose up -d gateway-service user-service order-service

# 第三步：启动 Jenkins（可选）
docker-compose up -d jenkins
```

### 4.3 停止服务

```bash
# 停止所有服务
docker-compose down

# 停止并删除数据卷（慎用，会清除数据库数据）
docker-compose down -v
```

## 六、推送代码到 GitHub

Jenkins 流水线需要从 GitHub 拉取代码，因此需要先将项目推送到 GitHub。

### 6.1 创建 GitHub 仓库

1. 登录 https://github.com
2. 点击右上角 "+" -> "New repository"
3. 填写仓库名，如 `microservice-jenkins-docker-demo`
4. 选择 Public 或 Private
5. **不要**勾选 "Add a README file"（保持空仓库）
6. 点击 "Create repository"

### 6.2 推送项目到 GitHub

```bash
cd 微服务Jenkins-Docker项目实践

# 初始化 Git 仓库
git init

# 添加所有文件
git add .

# 提交
git commit -m "初始提交：微服务 Jenkins Docker 部署项目"

# 关联远程仓库（替换为你的仓库地址）
git remote add origin https://github.com/你的用户名/你的仓库名.git

# 推送
git branch -M main
git push -u origin main
```

### 6.3 生成 GitHub Access Token（用于 Jenkins）

1. 登录 GitHub -> Settings -> Developer settings -> Personal access tokens -> Tokens (classic)
2. 点击 "Generate new token (classic)"
3. 勾选权限：`repo`（完整仓库访问）
4. 点击 "Generate token"
5. **复制保存** Token（只显示一次）

## 七、Jenkins 配置

### 7.1 安装必要插件

登录 Jenkins 后，进入 "Manage Jenkins" -> "Manage Plugins"，安装以下插件：

- **Docker Pipeline** - Docker 构建支持
- **Pipeline** - 流水线支持
- **Git** - 代码拉取

### 7.2 配置 GitHub 凭据

进入 "Manage Jenkins" -> "Manage Credentials"，添加 GitHub 凭据：

- Scope: Global
- Kind: Username with password
- ID: github-credentials
- Username: 你的 GitHub 用户名
- Password: 你的 GitHub Access Token

### 7.3 配置 Docker Hub 凭据

进入 "Manage Jenkins" -> "Manage Credentials"，添加 Docker Hub 凭据：

- Scope: Global
- Kind: Username with password
- ID: docker-hub-credentials
- Username: 你的 Docker Hub 用户名
- Password: 你的 Docker Hub 密码或 Token

### 7.4 创建流水线任务

1. 点击 "New Item"
2. 输入任务名称，如 `microservice-pipeline`
3. 选择 "Pipeline" 类型
4. 点击 "OK"

### 7.5 配置流水线

在任务配置页面：

1. **Pipeline** 部分：
   - Definition: `Pipeline script from SCM`
   - SCM: `Git`
   - Repository URL: 你的 Git 仓库地址
   - Branch Specifier: `*/main`
   - Script Path: `Jenkinsfile`

2. 或者直接使用 **Pipeline script**，粘贴 `Jenkinsfile` 内容

### 7.6 运行流水线

点击 "Build Now" 开始执行流水线。

流水线阶段：
1. **Checkout** - 拉取代码
2. **Build** - 编译项目
3. **Test** - 运行测试
4. **Docker Build** - 构建 Docker 镜像
5. **Docker Push** - 推送镜像到仓库
6. **Deploy** - 部署服务

### 7.7 使用共享流水线

在 Jenkins 中配置共享库：

1. 进入 "Manage Jenkins" -> "Configure System"
2. 找到 "Global Pipeline Libraries"
3. 添加库：
   - Name: microservice-library
   - Default version: main
   - Source: Git
   - Repository URL: 你的仓库地址

4. 在 Jenkinsfile 中使用：

```groovy
@Library('microservice-library') _

pipeline {
    agent any
    stages {
        stage('Build') {
            steps {
                script {
                    microservicePipeline.buildService('user-service')
                }
            }
        }
    }
}
```

## 八、Nacos 配置管理

### 8.1 导入配置文件

登录 Nacos 控制台，进入 "配置管理" -> "配置列表"：

1. 创建命名空间（可选）：`dev`
2. 点击 "+" 创建配置：
   - Data ID: `gateway-service.yml`
   - Group: `DEFAULT_GROUP`
   - 配置格式: YAML
   - 配置内容: 复制 `config/nacos/gateway-service.yml` 的内容

3. 重复创建 `user-service.yml` 和 `order-service.yml`

### 8.2 配置热更新

修改 Nacos 中的配置后，服务会自动刷新，无需重启。

## 九、常见问题排查

### 9.1 Nacos 启动失败

```bash
# 检查 Nacos 日志
docker-compose logs nacos

# 常见原因：
# 1. 端口被占用 - 检查 8848/9848/9849 端口
# 2. 内存不足 - 调整 JVM 参数
# 3. MySQL 未就绪 - 确保 MySQL 先启动
```

### 9.2 MySQL 初始化失败

```bash
# 检查 MySQL 日志
docker-compose logs mysql

# 如果数据卷已存在，初始化脚本不会执行
# 解决方案：删除数据卷重新创建
docker-compose down -v
docker-compose up -d mysql
```

### 9.3 服务注册失败

```bash
# 检查微服务日志
docker-compose logs user-service

# 常见原因：
# 1. Nacos 地址配置错误 - 检查 application.yml
# 2. 网络不通 - 检查 Docker 网络
# 3. 版本不兼容 - 确认 Spring Cloud Alibaba 版本
```

### 9.4 Feign 调用失败

```bash
# 检查 order-service 日志
docker-compose logs order-service

# 常见原因：
# 1. user-service 未注册到 Nacos
# 2. Feign 超时配置不当
# 3. 网络问题

# 测试 user-service 是否可达
docker exec order-service curl http://user-service:8081/actuator/health
```

### 9.5 Gateway 路由不生效

```bash
# 检查 gateway-service 日志
docker-compose logs gateway-service

# 常见原因：
# 1. Nacos 配置未导入
# 2. 路由配置格式错误
# 3. lb:// 协议需要 LoadBalancer 依赖

# 验证路由配置
curl http://localhost:8080/actuator/gateway/routes
```

### 9.6 Jenkins Docker 权限问题

```bash
# Jenkins 容器需要访问 Docker socket
# 确保 docker-compose.yml 中已挂载：
# - /var/run/docker.sock:/var/run/docker.sock

# 测试 Jenkins 容器内 Docker 命令
docker exec jenkins docker ps
```

## 十、重置环境

```bash
# 完全重置（删除所有容器、数据卷、网络）
docker-compose down -v --remove-orphans
docker-compose rm -f

# 重新构建并启动
docker-compose build
docker-compose up -d
```

## 十一、生产环境建议

1. **安全**:
   - 修改 Nacos、MySQL 默认密码
   - 配置 SSL/TLS
   - 限制 Docker socket 访问权限
   - 使用 Docker Secret 管理敏感信息

2. **高可用**:
   - Nacos 集群部署
   - MySQL 主从复制
   - 多副本微服务

3. **监控**:
   - 集成 Prometheus + Grafana
   - 配置日志收集（ELK）
   - 链路追踪（SkyWalking）

4. **CI/CD**:
   - 配置代码质量扫描
   - 自动化测试
   - 蓝绿部署 / 金丝雀发布
